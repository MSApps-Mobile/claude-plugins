# GitHub Presence Analysis Guide

## Profile Review

### Profile README (username/username repo)
- Does it exist? (many developers skip this)
- Does it communicate who you are and what you do?
- Elements to check:
  - Introduction / elevator pitch
  - Current focus / what you're working on
  - Tech stack / skills (badges or text)
  - Links to other platforms (website, LinkedIn, Twitter)
  - Contact information
  - Dynamic elements (GitHub stats, recent activity, etc.)

### Bio & Details
- Bio filled in? (160 char limit)
- Location, company, website links present?
- Profile photo (not default avatar)?
- Pinned repositories — are they the best showcase?

## Activity Analysis

### Contribution Graph
- Is it mostly green or mostly grey?
- Consistent activity or sporadic bursts?
- **Healthy pattern:** Regular commits across the week
- **Red flag:** Long gaps followed by huge dumps, or only activity during work hours on private repos

### Commit Patterns
- **Frequency:** Weekly active? Monthly? Quarterly?
- **Quality:** Meaningful commits or "update README" noise?
- **Message quality:** Descriptive or "fix stuff" / "asdf"?

### Repository Health
For pinned / highlighted repos, check:

| Aspect | Good | Needs Work |
|--------|------|------------|
| README | Clear install, usage, contributing sections | Missing or bare |
| License | Present and appropriate | Missing |
| Stars/Forks | Growing, even slowly | Zero after months |
| Issues | Managed (labeled, responded to) | Ignored or overwhelming |
| Last commit | Within 30 days | 6+ months ago |
| CI/CD | Badges showing, tests passing | No automation |
| Description | Clear one-liner | Empty |
| Topics/tags | 3-5 relevant tags | None |

### Open Source Presence Score

**Tier 1 (1-3/10):** Has a GitHub account, a few repos, mostly inactive
**Tier 2 (4-5/10):** Regular commits, a few repos with READMEs, some stars
**Tier 3 (6-7/10):** Active contributor, good READMEs, profile README, some community engagement
**Tier 4 (8-9/10):** Well-known repos, regular contributions to other projects, strong documentation
**Tier 5 (10/10):** Maintainer of popular projects, consistent community leader, exemplary documentation

## Strategic Assessment
- Does the GitHub activity support their professional brand?
- Would a potential employer/client be impressed visiting this profile?
- Is the tech stack visible and aligned with their claimed expertise?
- Are there any repos that should be made private? (old, broken, embarrassing)
- Is there a clear "portfolio" path? (pinned repos tell a story)
